  ```
  CTS = (referral_proximity × 0.25) +
        (estate_endorsement × 0.20) +
        (completion_rate × 0.15) +
        (recency_activity × 0.10) +
        (rating_score × 0.10) +
        (geo_proximity × 0.10) +
        (verification_tier × 0.05) +
        (job_volume_signal × 0.05)
  ```
- Trip lifecycle state machine (requested → matched → en_route → active → completed)

## CRITICAL NIGERIAN MARKET CONSTRAINTS FOR ARCHITECTURE

### 1. Connectivity Resilience
```
- Assume 3G connectivity on driver devices in satellite towns
- All API calls must have retry logic (exponential backoff)
- Driver app: job queue must survive app restart (AsyncStorage persistence)
- Passenger app: show cached map tiles, last known driver positions
- API timeout: 10s max, with meaningful error messages in Pidgin
- SMS fallback: if FCM fails, send Termii SMS for trip request
```

### 2. Data Efficiency
```
- Driver location updates: compress to {lat, lng, b (bearing)} = 50 bytes/update
- Location update frequency: every 3 seconds while trip active, every 10s while online/idle
- Map tiles: cache aggressively (MapType.ROADMAP, Nigerian road network)
- Image optimization: compress KYC documents to 800KB max before S3 upload
- API response pagination: max 20 items per page, cursor-based
```

### 3. Geofencing (LEGALLY REQUIRED)
```javascript

### 4. Payment Architecture (Paystack Nigeria)
```
Flow:
1. Passenger books trip → Paystack.initialize() → charge card/USSD
2. Payment authorized → held as "pending disbursement"
3. Trip completes → platform fee deducted → Paystack.transfer() to driver's bank
4. Driver subscription → Paystack recurring → monthly/weekly/daily auto-charge

Important:
- Platform commission: 10% (Year 1), 12% (Year 2+)
- All amounts in kobo (₦1 = 100 kobo) in Paystack API
- Webhook must be idempotent (Paystack may retry)
- Driver bank details stored encrypted (AES-256)
- CBN regulation: platform does NOT hold wallet at this stage (no PSSP licence yet)
- PSSP licence application planned for Month 12 — architecture must accommodate wallet later
```

### 5. AI/ML Layer Architecture
```
Phase 1 (build now, low complexity):
- Document OCR: AWS Textract → extract text → validate → auto-fill KYC form
- Daily earnings summary: NestJS service calls OpenAI/Google → generates Pidgin summary
- Trivia question bank: static JSON, 200 questions, Nigerian culture/history/geography

Phase 2 (architect stubs now, implement later):
- Voice commands: Google Speech-to-Text → parse intent → execute
- Demand prediction: booking frequency time-series → alert drivers
- AI matching enhancement: provider embedding scores
- Skill assessment: vision model for driver portfolio photos

Stub pattern: create service interface with placeholder implementation
  Example: VoiceService.processCommand(audioBuffer) → returns MockCommandResult
  Real implementation swapped in Phase 2 without breaking contracts
```

## HICONNECT COMMUNICATION ENGINE (HCE) — MANDATORY MODULE
## This is not optional. Wire into every relevant agent.

### Why This Exists
70%+ of HiGo's target driver base cannot read English fluently.
The HCE removes four barriers: reading, writing, language, technology literacy.
It is HiGo's deepest competitive moat and the shared infrastructure for all future Hiconnect products.
